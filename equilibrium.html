The equilibrium IDS is a container object that has a single data object, a double(:) with all the timeslice values for scalar data. The equilibrium IDS also holds 4 container objects with data, or meta data, or other containers: ids_properties, code, vacuum_toroidal_field and time_slice(i1), respectively. Each time_slice(i1) container holds a single data object, a scalar with the time value, and the 6 containers global_quantities, boundary, profiles_1d, profiles_2d(i2), coordinate_system, ggd(i2). global_quantities has the containers magnetic_axis and q_min. boundary has the containers geometric_axis, outline, lcfs, x_point(i2), strike_point(i2) and active_limiter_point. profiles_2d(i2) has the containers grid_type and grid. coordinate_system also has containers grid_type and grid. ggd(i2) has containers r(i3), z(i3), psi(i3), phi(i3), theta(i3), j_tor(i3), j_parallel(i3), b_field_r(i3), b_field_z(i3), b_field_tor(i3) and grid. grid has containers identifier, space(i3) and grid_subset(i3). space(i3) has containers geometry_type and objects_per_dimension(i4), which has a container object(i5), which has a container boundary(i6). grid_subset(i3) has containers identifier, base(i4), metric and element(i4), which has object(i5).

The equilibrium IDS has a total of 42 container objects with 632 data (or meta data) objects.
